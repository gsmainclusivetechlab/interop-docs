### Creating a Session

1. [**Select** the function **New Session**](session#step-01-sut-selection)
2. [**Select** the System Under Test (**SUT**) and add its **type** and **URL**](session#step-02-session-info)
3. [**Add** the **Session Information**](session#step-02-session-info)
4. [**Choose** the **Test Cases** for the session](session#step-02-session-info)
5. [**Get** the **Configuration Components**](session#step-03-configure-components)
6. [**Check out** the **Created Session**](session#)

### Running Tests

1. [**Get** the **Postman Collections** with the preconfigured test requests](tests#postman-collections)
2. [**Use** the Postman or other **API Client Tool** to run the tests](tests#sending-requests-from-postman)
3. [**Set** the **Environment and Variables** for your test session](tests#setting-test-environment)
4. [**Send** a **Test Request** to the Interoperability Test Platform](tests#sending-a-test-request)
5. [If you prefer, **Use** the **Collection Runner** for multiple requests](tests#running-multiple-tests)

### Test Results

1. [**Open** the **Tested Session** in the Interoperability Test Platform](results#session-results)
2. [**Access** the **Test Case** of your choice](results#accessing-test-case-results)
3. [**Check** the **Test Information**](results#test-information)
4. [**Analyse** the **Test Run** Information to see it pass or fail and why](results#test-runs)
5. [**Check** the **Test Steps** to see all performed tests for each step](results#test-steps)
6. [**Access** the **Test Flow** to see a sequence diagram with the messages between the steps](results#test-flow)
